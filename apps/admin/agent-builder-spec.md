**Core principle:** Anyone can build an agent. The marketing manager, the CEO, the ops lead. Not just developers.

## UX Principles

1. **Progressive disclosure** â€” Start simple, reveal complexity as needed
2. **Sensible defaults** â€” Department selection pre-populates relevant tools
3. **Test before publish** â€” Can't go live without running at least one test
4. **Version everything** â€” Every publish creates a version, easy rollback
5. **Show the learning** â€” Display teaching count, rating, learned rules
6. **Inline help** â€” Tool descriptions, guidance text everywhere
7. **Visual feedback** â€” Show what's selected, what's dangerous, what's required

## Access Control

| Role | Can Do |
|------|--------|
| Admin | Full access, publish, delete |
| Editor | Create, edit, test (not publish) |
| Viewer | View only |

Enterprise tier can restrict:
- Which departments users can create agents for
- Which tools are available
- Maximum agent count

### Agent Builder â†’ Agent Runtime
- Publish generates Agent SDK config

- Runtime loads config on agent invocation

- Test tab captures feedback
- "Edit & Teach" opens teaching mode
- Teachings flow to Admin dashboard

- Inline skill editor
